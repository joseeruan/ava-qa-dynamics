## Agent Customization File

### Customization Choice

User chose to create customization file for experimental flexibility and future tweaking.

### Customization Purpose

The customization file provides an easy way to adjust Marcos's behavior without modifying core agent files. It acts as a settings overlay that allows safe experimentation with:

- **Personality tweaks**: Adjust communication style or specific guiding principles
- **Command modifications**: Add new commands or modify existing command behaviors
- **Experimental features**: Try new approaches with quick settings toggles
- **Framework preferences**: Override default testing framework and library choices
- **Custom variables**: Define project-specific defaults for test generation

### File Location

`{project-root}/.bmad/_cfg/agents/custom-dynamics-qa-expert.customize.yaml`

### Usage Guidance

**Getting Started:**
1. Open the customization file in your editor
2. Review the commented examples for common customization scenarios
3. Uncomment (remove `#`) the sections you want to customize
4. Modify values to suit your preferences
5. Save and restart Marcos for changes to take effect

**Safety Features:**
- Core agent files remain untouched and stable
- Invalid configurations are safely ignored with warnings
- You can experiment freely without breaking the agent
- Easy to revert by commenting out or deleting changes
